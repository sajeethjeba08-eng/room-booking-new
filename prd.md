PRD — Affordable Room Booking Website (OYO Style Demo)
1. App Overview & Objectives
Overview
An OYO style room booking website demo that emphasizes clear affordability and fast booking. The product showcases a complete end to end booking flow (search → select → confirm) while deliberately excluding complex features like payments and user accounts.
Primary Objective
Demonstrate strong product thinking and UX clarity by showing how a user can confidently book an affordable room in just a few steps.
Success Criteria
•	Users can complete a booking flow without confusion or friction
•	Affordability is immediately obvious through bold offers and discounts
•	Recruiters can clearly see prioritization, scope control, and user centric decision making
Non Goals (Out of Scope)
•	Payments and refunds
•	User authentication or profiles
•	Reviews, ratings, loyalty programs
•	Real time inventory or multi city search
________________________________________
2. Target Audience
Primary User: Traveler
•	Context: Booking a short stay online
•	Behavior: Price sensitive, wants quick decisions
•	Needs: Fast booking, visible discounts, no hidden complexity
Secondary Audience: Recruiters / Evaluators
•	Goal: Assess product thinking, UX flow, and prioritization
•	Expectation: Clear rationale behind feature inclusion and exclusion
________________________________________
3. Core Value Proposition
“Book an affordable room quickly, with clear deals and zero confusion.”
Key differentiator is perceived affordability, not feature depth.
________________________________________
4. Core User Flow (Happy Path)
Start: User lands on homepage
1.	User sees bold offer messaging (e.g., “Up to 60% OFF on Rooms”)
2.	User enters location, dates, and number of guests
3.	User clicks Search
4.	System displays a list of available rooms with visible discounts
5.	User selects a room
6.	User reviews room details and offer breakdown
7.	User clicks Book Now
8.	System shows booking confirmation with savings highlighted
End: User feels happy and confident about getting a good deal
________________________________________
5. Core Features & Functionality
ID	Feature	Description
F1	Search Input	Location, dates, guests
F2	Room Listings	Static/mock data with discounted prices
F3	Room Details	Images, amenities, price, discount info
F4	Booking Action	One click booking (no payment)
F5	Confirmation	Summary + savings reinforcement
________________________________________
6. User Experience & Interface Design
6.1 Homepage
•	Hero banner with bold discount messaging
•	Minimal copy focused on savings
•	Prominent search bar
6.2 Search Results Page
•	Room cards showing:
o	Discounted price (large)
o	Original price (strikethrough)
o	Offer badge (e.g., “50% OFF”)
•	Simple sorting (optional: “Best Deal”)
6.3 Room Details Page
•	Reinforces affordability
•	Clear breakdown of savings
•	Strong Book Now CTA
6.4 Confirmation Page
•	Positive reinforcement messaging
•	Booking summary
•	Highlight: “You saved ₹X on this booking”
________________________________________
7. Feedback, States & Error Handling
System States
•	Loading: “Searching rooms…” spinner
•	Success: Instant display of room list
•	Empty State: “No rooms available for selected dates”
Error Handling (Minimal)
•	Missing inputs → Prompt user to complete fields
•	System failure → Retry option
________________________________________
8. Data & Logic (High Level)
•	Inputs: User search criteria
•	Data Source: Static or mocked room inventory
•	Logic: Filter rooms → display matching results
•	Persistence: None (UI only demo)
________________________________________
9. Security & Trust Considerations
Even as a demo, the app should:
•	Avoid misleading pricing language
•	Clearly indicate demo nature if needed
•	Display consistent pricing throughout the flow
________________________________________
10. Potential Challenges & Mitigations
Challenge	Mitigation
Offers feel fake	Keep pricing consistent and simple
UI feels cluttered	Limit number of offer elements per screen
Demo scope creep	Strictly enforce non goals

11. Future Expansion Possibilities

12. Summary
This demo product intentionally focuses on affordability clarity and booking simplicity. By prioritizing bold offers and a frictionless flow, it communicates both strong user empathy and disciplined product decision making.
